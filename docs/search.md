Searches posts matching a SINGLE query. This command is ONLY for single-term or single-operator searches (e.g. one keyword, `from:username`, a single `"exact phrase"`). If you have 2 or more search terms, do NOT use this command — use `batch-search` instead, which correctly joins terms with the OR operator. Most search tasks should use `batch-search`. By default searches the last 7 days (GET /2/tweets/search/recent). With `--all`, searches the full archive back to 2006 (GET /2/tweets/search/all). Invoke via `node <base_directory>/x.js search "<query>" [flags]`. Output is JSON to stdout.

[!FLAGS] a) no flags — returns up to 10 results with default tweet fields (text, author_id, created_at, conversation_id, public_metrics) and expands author (name, username). b) `--all` — search the full archive instead of just the last 7 days (requires `X_API_BEARER_TOKEN` to be set; the client auto-selects Bearer auth for this endpoint). c) `--max-results <10-100>` — set number of results per page (up to 500 with --all). d) `--sort <recency|relevancy>` — order results. e) `--start-time <ISO8601>` — oldest timestamp (inclusive). f) `--end-time <ISO8601>` — newest timestamp (exclusive). g) `--since-id <id>` — results newer than this post ID. h) `--until-id <id>` — results older than this post ID. i) `--next-token <token>` — pagination token from a previous response. j) `--fields <comma-list>` — override default tweet fields. k) `--raw` — output the full API response envelope (data, includes, meta, errors).

[!QUERY-SYNTAX] The query uses X search operators. Examples: `from:username`, `has:media`, `-is:retweet`, `#hashtag`, `"exact phrase"`, `lang:en`. Combine with AND (space), OR, NOT (-). Max 4096 chars. Full reference: https://docs.x.com/x-api/posts/search/integrate/build-a-query.

[!WRONG-TOOL-WARNING] This command accepts a SINGLE query string and sends it verbatim to the X API. Do NOT pass comma-separated terms here — commas are literal characters in X search syntax, not separators. A query like `"term1,term2,term3"` searches for the exact string "term1,term2,term3", which is almost certainly not what you want. For multi-term searches, ALWAYS use `batch-search --terms "term1,term2,..."` instead, which correctly joins terms with the `OR` operator.

[!OUTPUT-SHAPE] Default produces an array of tweet objects. With `--raw`, wraps into the API envelope with `data` (tweets), `includes` (expanded users), and `meta` (result_count, next_token for pagination).
